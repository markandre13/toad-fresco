WidgetKit [index]

declared in file: /opt/Fresco/Fresco/defs/widgets.idl

WidgetKit provides operations for creating common types of buttons, and scrolling objects, as well as styled frames and labels.

Glyph push_button_frame(in Glyph g, in Telltale t, in Coord thickness,in float xalign, in float yalign);
Glyph filled_push_button_frame(in Glyph g, in Telltale t, in Coord thickness,in float xalign, in float yalign);
These operations provide a beveled frame around the given glyph. The inset_frame operation uses dark shading in the upper left and light shading in the lower right to make the contents of the frame appear to be recessed. The outset_frame operation reverses the shading to make the contents appear to project out of the frame. The bright_inset_frame operation is like inset_frame but uses a brighter background color and is a thinner frame. The hilite_frame operation dynamically selects the type of frame based on the telltale state. The push_button_frame creates a similar telltale driven frame that looks like the beveled frame of a push button.

FieldEditor field_editor(in FieldSubject subj, in long width, in long maximum);
FieldEditor managed_field_editor(in ExclusiveEditMgr mgr,in Tag subj, in FieldEditorData fe_data);
The field_editor operation returns a field editor used to input and edit a string of text. The subj argument is a FieldSubject which will be notified when the field editor commits changes. (By default, changes are committed when the user presses the Enter or Esc keys.) If the subj argument is nil, then then a subject will be created for it. The width argument, specified in characters, determines the natural size of the text field. The maximum argument, also specified in characters, limits the number of characters the field editor will allow during input. If maximum is set to zero, then the user is allowed to input any number of characters. . The managed_field_editor returns a field_editor edited under the control of a exclusive edit manager.

Return a button that frames the given glyph and executes the given action when clicked. A default button adds some decoration to indicate to the user that it is the common choice. Unlike a push or default button, a palette button can be chosen. A transparent button is like a push button without any of the decoration. a palette button can be chosen.

Return a check box with the check to the left of the given glyph. A check box is a toggle button: choosing it when already chosen will cause it to become unchosen.

Return a radio button with the given glyph to the right of the flag that indicates whether the button is chosen. A radio button has a telltale group so that only one button in a group will be chosen at any given time.

Return a scroll bar displayed along the given axis that modifies the given adjustment. A scroll bar is just a slider with movers on either side that adjust in fixed increments.
Viewer panner(in Adjustment x, in Adjustment y);
Return a panner that is displayed as a rectangular thumb in a two-dimensional area. The thumb's X position adjusts the first adjustment parameter; the thumb's y position adjusts the second parameter.
Button up_mover(in Adjustment a);
Button down_mover(in Adjustment a);
Button left_mover(in Adjustment a);
Button right_mover(in Adjustment a);
Movers are buttons that modify an adjustment by a fixed amount. The buttons show an arrow pointing up, down, left, or right. The right and down movers increment the adjustment's current setting. The left and up movers decrement the setting.

Return a viewer with the given glyph as its body. The viewer may contain other viewers and will interpret tab to move focus in child order.
