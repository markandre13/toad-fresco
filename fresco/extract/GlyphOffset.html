GlyphOffset [index]

declared in file: /opt/Fresco/Fresco/defs/glyph.idl
ancestors: FrescoObject

A glyph offset is a reference to a glyph within an aggregate. Because a glyph might appear twice within the same composite, a glyph itself is insufficient to identify its position within its parent.
Glyph parent();
Glyph child();
Return the composite parent and child glyph to which the offset refers.
GlyphOffset next_child();
GlyphOffset prev_child();
Return the next or previous offset within the list of children.
void allocations(inout Glyph::AllocationInfoSeq a);
Compute cumulative information for each allocation of the glyph. This operation normally need not be called directly; it is normally called by an implementation Glyph::allocations.
void insert(in Glyph g);
Insert the given glyph into the composite before the glyph referred to by the target offset.
void replace(in Glyph g);
Put the given glyph in place of the glyph at this offset. The replace operation is equivalent to an atomic transaction that removes the old glyph and inserts the new one.
void remove();
Take the glyph at this offset out of the composite. The offset will now refer to the following glyph. If the offset refers to the end of the composite, then this operation will have no effect.
void notify();
Notify the composite that the desired size of the glyph at this offset has changed.
void traverse(in GlyphTraversal t);
Traverse along to the child glyph. Normally, this operation simply would call child->traverse(t), but in some cases the parent may want to use a child-specific traversal operation.
void visit_trail(in GlyphTraversal t);
Notify the composite that the desired size of the glyph at this offset has changed.
void child_allocate(inout Glyph::AllocationInfo a);
Determine the allocation for the child given the allocation information for the parent.
