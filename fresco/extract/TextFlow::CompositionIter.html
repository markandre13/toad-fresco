TextFlow::CompositionIter [index]

declared in file: /opt/Fresco/Fresco/defs/text.idl

A CompositionIter allows one to navigate over a composition. An iterator gives relatively efficient access to the structure of the Composition. The iterator can be positioned in both dimensions. Changes made to the Composition may cause the iterator to become invalid.
void next();
void prev();
boolean done();
boolean is_first();
boolean is_last();
void reset_to_first();
void reset_to_last();
These operations are based upon the current line the iterator is positioned over. The next and previous functions move to the next and previous offset respectively within the current line. The done function determines if the iterator is still positioned over the line. The is_first and is_last functions determine if the iterator is positioned over an endpoint of the current line. The reset_to_first and reset_to_last functions place the iterator at an endpoint.
Glyph get_glyph();
CompositionGlyphOffset get_offset();
Glyph line_glyph();
long line();
The function get_glyph gives access to the current child glyph that the iterator is positioned over. The get_offset operation returns the current child offset that the iterator is positioned over. The line_glyph operation gives the glyph that represents the entire line that the iterator is positioned over. The line operation returns the line number of the line currently positioned over.
Coord get_ascent();
Coord get_descent();
Coord height();
Coord get_span();
The get_ascent and get_descent operations return the height above and below the baseline respectively. The height operation returns the total height of the line including separation from the next line. The get_span operation returns the span of the offset the iterator is currently positioned over along the major axis (the width).
void next_line();
void prev_line();
boolean valid();
boolean forced_break();
Move the iterator's position to the start of the next/previous line respectively. The valid function determines if the iterator is positioned over a line or not. Even if valid, the iterator may not be positioned over a glyph... a call to reset_to_first() or reset_to_last() may be required to do that (and done() is the associated test). The forced_break() function determines if the line ends in a forced break, which might be useful for things like determining logical line numbers.
