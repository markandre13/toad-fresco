TextFlow::Breakable [index]

declared in file: /opt/Fresco/Fresco/defs/text.idl

A Breakable is information for use in determining where to create breaks. A Breakable defines multiple looks for use under different break conditions. The Breakable behavior may be either static or dynamic (ie. returning shared glyphs or creating glyphs on the fly for different break conditions).
void broken_look(in Axis a, in Coord len, out Glyph pre, out Glyph sep, out Glyph post,out Breakable br_pre, out Breakable br_sep, out Breakable br_post);
The various possible looks, depending upon the break condition. A Breakable implementing static breaking behavior may ignore the len argument which indicates where the desired break should occur.
const long penalty_good = -10000;
const long penalty_bad = 10000;
These are the bounds for penalties used for determining breaks. A good penalty value should always be less than a bad penalty value. The penalty_good value is a value that is considered a forced break (ie. a value <= this value will be guaranteed to cause a break). The penalty_bad value is a value that is considered to be unbreakable (ie. a value >= this value will be guaranteed not to break).
void penalty(in Axis a, in Coord len, out long break_penalty,out long resize_penalty);
boolean is_forced_break(in Axis a);
The penalty function determines the penalty for taking a break. The len argument specifies where a potential break is desired within the span of the glyphs unbroken requirements. Breakable's with dynamic breaking behavior (ie those that have different broken looks depending upon where the break is requested) are not allowed to span across a forced break, so a length is not needed to determine if a breakable describes a forced break. . The size_penalty function determines the penalty for changing the size of the breakable if it supports multiple sizes. Some compositors may look at this information and use it to determine whether to break or resize based upon it. Other simpler compositors may ignore this value and enforce their own policy of break versus resize.
